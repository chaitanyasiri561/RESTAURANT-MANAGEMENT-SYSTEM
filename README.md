# 🍽️ Restaurant Management System (DBMS Project)

## 📌 Overview

This project is a Database Management System (DBMS)-based Restaurant Management System designed to efficiently manage restaurant operations such as customers, orders, chefs, servers, and staff.
It replaces manual processes with a structured database system to improve accuracy, reduce redundancy, and enhance overall efficiency.

---

## 🚀 Features

* ✔️ Customer Management
* ✔️ Order Processing System
* ✔️ Staff, Chef & Server Management
* ✔️ Efficient Data Storage using Relational Database
* ✔️ SQL Queries (Joins, Subqueries, Aggregate Functions)
* ✔️ Views for Simplified Data Access
* ✔️ Database Normalization up to 3NF

---

## 🛠️ Tech Stack

* **Database:** SQL (MySQL / SQLite)
* **Concepts Used:** DBMS, ER Model, Normalization

---

## 🗂️ Database Design

The system is designed using multiple related tables:

* **Restaurant** – Stores restaurant details
* **Staff** – Contains employee information
* **Chefs** – Stores chef details and specialization
* **Servers** – Manages serving staff and table assignments
* **Customer** – Stores customer data
* **Orders** – Maintains order records and relationships

---

## 🧠 Concepts Implemented

* Entity-Relationship (ER) Model
* Primary Keys & Foreign Keys
* Normalization (1NF, 2NF, 3NF)
* SQL Joins
* Subqueries
* Aggregate Functions
* Views
